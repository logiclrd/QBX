So, if an error comes from the program, use the error code directly. If a runtime exception results
from something the IDE does, add 2000 before negating.

and -2025 from getting an error back

-24 and -25 come from LPRINT

-71 if disk isn't ready from action taken by the code
-2071 if disk isn't ready when trying to save/load within the IDE

-72 if disk I/O error from code
-2072 if disk I/O error from IDE action

-76 if path not found from code
-2076 if path not found from IDE action
