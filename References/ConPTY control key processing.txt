Client side:

WM_KEYDOWN/WM_KEYUP handlers in TerminalContainer_MessageHook in TerminalContainer.cs
-> TerminalSendKeyEvent
-> HwndTerminal::_SendKeyEvent
-> Terminal::SendKeyEvent
{
  receives { isKeyDown, VK, scanCode, character }, builds INPUT_RECORD structure
}
-> TerminalInput::HandleKey
-> _makeWin32Output OR _encodeKitty OR _encodeRegular


Server side:

VtInputThread::_InputThread
-> StateMachine::ProcessString
{
  calls FindActionableControlCharacter
  calls _ActionPrintString on non-control-characters at the front of the string
  calls ProcessCharacter for each remaining character in the string
}
-> StateMachine::ProcessCharacter
{
  checks for C1 control characters
  checks for ESC introducing escape sequences
  dispatches the event based on the state
}
-> StateMachine::_EventGround
{
  checks for C0 control characters
}
-> StateMachine::_ActionExecute
{
  calls ActionExecute on an injected implementation of IStateMachineEngine
}
-> InputStateMachineEngine::ActionExecute
-> InputStateMachineEngine::_DoControlCharacter
{
  checks for C0 control characters
  calls _GenerateKeyFromChar
}
-> InputStateMachineEngine::_WriteSingleKey
{
  calls WriteCtrlKey on an injected implementation of IInteractDispatch
}
-> InteractDispatch::WriteCtrlKey
-> HandleGenericKeyEvent
-> HandleCtrlEvent
{
  sets a flag in the CtrlFlags member of CONSOLE_INFORMATION
}

then later, when unwinding the call stack, the console is unlocked:

UnlockConsole
-> ProcessCtrlEvents
{
  uses a service locator to find an implementation of IConsoleControl
  I am assuming the call then flows into the Win32-specific implementation,
  which seems to be supported by observations with a debugger attached
}
-> ConsoleControl::EndTask(process ID, e.g. CTRL_C_EVENT, CtrlFlags)
{
  populates a CONSOLEENDTASK structure
}
-> ConsoleControl::Control(ControlType::ConsoleEndTask, &params, sizeof)
-> kernel API: ConsoleControl(command, ptr, len)

I don't have access to the source code of ConsoleControl, I think exported
by user32.dll, but a glance at its disassembly suggests that it hands off
the request to csrss.exe using LPC
